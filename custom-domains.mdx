---
title: "Custom domains"
description: "Serve a Frame under your own hostname"
---

Every Frame gets a Nubo-assigned domain out of the box. When you're ready to go live you can attach a custom domain — anything like `shop.mystore.com`, `api.mysite.io`, or even a root domain — and Nubo will serve the Frame there with HTTPS.

The exact DNS you'll set up depends on whether the Frame runs on **managed** infrastructure (our cloud) or a **self-hosted agent** (your own server). The API is the same either way.

## Add a domain

```bash
curl -X POST https://shuttle.withnubo.com/v2/projects/<project_id>/spaces/<space_id>/frames/<frame_id>/domains \
  -H "Authorization: Bearer <your-jwt>" \
  -H "Content-Type: application/json" \
  -d '{"hostname": "shop.mystore.com"}'
```

The response tells you exactly which DNS records to create. The shape differs by infrastructure.

### Managed Frames

Managed Frames use Cloudflare for SSL for SaaS, so you'll create two records — a CNAME for traffic and a TXT for domain validation:

```json
{
  "hostname": "shop.mystore.com",
  "dns": {
    "record_type": "CNAME",
    "record_name": "shop.mystore.com",
    "record_value": "proxy.withnubo.com"
  },
  "verification": {
    "record_type": "TXT",
    "record_name": "_cf-custom-hostname.shop.mystore.com",
    "record_value": "a1b2c3d4-..."
  }
}
```

Add both to your registrar. Then tell Nubo to verify:

```bash
curl -X POST https://shuttle.withnubo.com/v2/projects/<project_id>/spaces/<space_id>/frames/<frame_id>/domains/shop.mystore.com/verify \
  -H "Authorization: Bearer <your-jwt>"
```

If the TXT record is in place, Cloudflare will issue a certificate and Nubo will wire up the routing. You can call `/verify` again any time to re-poll.

### Self-hosted agent Frames

Agent Frames use the agent's built-in Caddy, which requests a Let's Encrypt cert automatically — no verification step:

```json
{
  "hostname": "shop.mystore.com",
  "dns": {
    "record_type": "CNAME",
    "record_name": "shop.mystore.com",
    "record_value": "abc123.frames.example.com"
  },
  "note": "Caddy on the agent will auto-provision TLS via ACME once the CNAME is in place."
}
```

Add the single CNAME, pointing at either the Frame's system subdomain or the agent's base domain — any name that resolves to the agent's IP works. The first HTTPS request to the new hostname triggers Caddy's ACME flow; after that, the Frame is live.

<Note>
  For agent Frames, port 80 must be reachable from the public internet so Caddy can complete the ACME HTTP-01 challenge. If it's firewalled, TLS provisioning will fail.
</Note>

## Verify it's working

Once DNS has propagated (usually seconds to a few minutes):

```bash
curl -vI https://shop.mystore.com
```

You should get a `200` back with a valid certificate. `dig shop.mystore.com` should show your CNAME resolving to the right target.

## Remove a domain

```bash
curl -X DELETE https://shuttle.withnubo.com/v2/projects/<project_id>/spaces/<space_id>/frames/<frame_id>/domains/shop.mystore.com \
  -H "Authorization: Bearer <your-jwt>"
```

This detaches the hostname from the Frame. For managed Frames it also removes the Cloudflare custom hostname; for agent Frames it reloads Caddy without the hostname. Remove the CNAME from your registrar once you're done.

## List attached domains

```bash
curl https://shuttle.withnubo.com/v2/projects/<project_id>/spaces/<space_id>/frames/<frame_id>/domains \
  -H "Authorization: Bearer <your-jwt>"
```

Returns the Frame's system domain(s) plus every custom domain attached, with their current status (`pending`, `active`, or `error`).

## Troubleshooting

### Status stuck on `pending`

- **Managed:** the TXT verification record isn't in place or hasn't propagated. Re-check your DNS and call `/verify` again.
- **Agent:** the agent was offline when the domain was added. Restart the agent — the domain applies on the next reconnect.

### TLS certificate error

- **DNS propagated?** `dig shop.mystore.com` should return the expected target.
- **Port 80 reachable?** (Agent Frames only.) Caddy needs an open port 80 for the ACME HTTP-01 challenge.
- **Rate-limited?** Let's Encrypt caps issuance per hostname per week. If you've retried a lot, wait an hour.

Agent logs show the full ACME exchange — check them with `journalctl -fu nubo-agent` (Linux) or `tail -f ~/Library/Logs/nubo-agent/stderr.log` (macOS).

### Hostname already registered

Each hostname belongs to one Frame at a time across all of Nubo. If you try to attach one that's already in use, the API returns a `400`. Remove it from the other Frame first, or pick a different hostname.
