title: "What are Agents?"
description: "Deploy Frames on your own infrastructure with Nubo Agents"

Nubo Agents let you run your applications on your own machines instead of our managed cloud. You get the same deployment experience (git push, build, deploy) but the workload runs on hardware you control.

## How it works

An agent is a lightweight binary that runs on your server. It connects back to Nubo and waits for work. When you deploy a Frame to an agent, the build and the runtime both happen on your machine.

Here's what happens when you deploy:

1. You create a Frame in Nubo and pick which agent should run it
2. Nubo sends the work down to your agent
3. The agent builds your app and starts it on your server
4. Your Frame is served at its own subdomain on your domain, or at a [custom domain](/custom-domains) you attach
5. Build logs stream back to the Nubo dashboard in real time

## When to use agents

Agents are a good fit when you:

- Need to run workloads on specific hardware (GPUs, high memory, etc.)
- Want to keep data on-premise for compliance reasons
- Already have servers and want to use them
- Want more control over the runtime environment

If none of that applies, our managed infrastructure works out of the box with no setup.

## Ownership and status

Agents are tied to your user account: they're owner-scoped, not shared across a Team or Organization. Each person registers and manages their own agents.

Every agent shows a live status of online or offline, which reflects whether it's currently connected to Nubo. An offline agent can't be chosen as a deploy target: its option is disabled in the New Frame **Target** picker until it reconnects.

## Removing an agent

You can remove an agent from the **Agents** section of **Settings**. If any Frames still target it, the removal is blocked and you'll see a message like `N frame(s) still target this agent; remove or migrate them first`. Delete or move those Frames to another target first, then remove the agent.

## What you'll need

- A Linux or macOS machine (the installer sets up the tools the agent needs: git, Docker, the Cloud Native Buildpacks pack CLI, and Caddy)
- A domain with a wildcard DNS record pointing to your server (for Frame subdomains)
- A Nubo account
